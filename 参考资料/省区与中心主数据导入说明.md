# 省区与中心主数据导入说明

## 一、用途

- 为**双重预防机制**（隐患上报、自查自纠、整改验收）提供组织主数据：全国**省区**与**转运中心**。
- 各省区、各中心可在本范围内自行上报隐患、自行整改；系统按省区/中心做数据隔离与统计。

## 二、数据来源

- **文件**：`公司通讯录（260319）.xlsx`（全网通讯录-直属，按日期更新）
- **Sheet**：首个 Sheet（如「全网通讯录（直属）3.19日更新」）
- **表头行**：第 3 行
- **数据行**：第 4 行起

## 三、字段映射（Excel → 系统）

| Excel 列名       | 说明           | 导入用途                 |
|------------------|----------------|--------------------------|
| 省公司/片区      | 省公司或片区   | 与「省区」一起做省区维度 |
| 省区             | 省区名称       | **省区主数据**、责任单位 |
| 分区名称         | 分区简称       | 省区展示/筛选            |
| 转运中心名称     | 中心全称       | **中心主数据**           |
| 简称             | 中心简称       | 中心展示、责任单位       |
| 类型             | 中心/其他      | 区分是否为中心           |
| 级别             | 一级/二级/三级 | 中心级别                 |
| 属性             | 直属等         | 中心属性                 |
| 经理/负责人      | 负责人姓名     | 中心负责人               |
| 电话             | 负责人电话     | 联系信息                 |
| 地址             | 中心地址       | 发生地点、场地信息       |
| 运营负责人       | 省区运营负责人 | 省区维度                 |

## 四、导入结果说明

- **省区**：从 Excel 中「省区」列去重，得到全国省区列表（约 23 个）。
- **中心**：每一行一条中心记录，关联所属省区；约 77 个中心（以当前通讯录为准）。
- 导入后用于：
  - 隐患上报时的「发生地点」：省区 + 中心 选择
  - 责任单位选择：省区、中心
  - 列表与报表按省区/中心筛选、统计

## 五、执行导入

在项目目录下执行：

```bash
# 依赖：openpyxl（若未安装：pip install openpyxl）
python3 scripts/import_provinces_centers.py
```

- 默认读取：`公司通讯录（260319）.xlsx`
- 输出：
  - `data/provinces.json`：省区列表
  - `data/centers.json`：中心列表（含省区、负责人、地址等）
  - `data/provinces_centers.csv`：省区与中心扁平表（便于入库或 Excel 二次处理）

更新通讯录后，替换 Excel 文件并重新执行脚本即可刷新主数据。
