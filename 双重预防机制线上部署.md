# 二、隐患排查治理

## 2.0 统一闭环口径（验收通过关闭）

最终关闭以“验收通过-关闭”为准；整改在“整改单”对象上闭环。

### 隐患记录状态（用于总部稽核循环）
- `待稽核`
- `稽核通过（已生成整改单）`
- `稽核不通过-待修正`
- `待再次稽核`

### 整改单状态（用于整改-验收闭环）
- `待整改`
- `整改中`
- `待验收`
- `验收通过-关闭`
- `验收不通过-退回整改`（回到 `整改中`，再次提交验收）

### 责任归属口径
- `责任单位`为流程归属对象（多选允许），默认粒度：`中心`。

## 2.1 隐患上报功能（日常自觉）

 
    
**页面清单**
- `隐患上报列表`：筛选（类别/地点/状态/时间）、列表展示、详情入口
- `新增隐患上报`：表单提交
- `隐患详情`：展示内容、附件、流转记录、可进入申诉中心（如适用）

**权限（按组织架构配置）**
- 省区安全员、中心安全员、总部人员、班组长等：可新增/可查看/可处理权限由系统配置

**新增隐患上报表单（建议字段）**
- `隐患类别`（下拉）：人员安全/场地安全/消防安全/用电安全/交通安全/设备安全/操作安全
- `二级隐患描述`（下拉）：二级模板选项
- `其他`选项：选择“其他”时显示 `自填隐患描述`（必填）
- `具体问题描述（自填）`（必填）
- `发生地点`（省区/中心/班组 + 场地/区域）（选填或必填按制度）
- `发现时间`（默认当前）（选填）
- `附件`（照片/文件，多选，可配置是否必填）

**校验规则（必须做）**
- `隐患类别`必填
- `具体问题描述`必填
- 若 `二级隐患描述=其他`：`自填隐患描述`必填

**提交后的默认流转**
- 状态置为：`待稽核`
- 进入总部稽核任务视图（2.3）


## 2.2 自查自纠（任务下发）

**配置中心：中心安全检查表模板（用于巡检项结构）**
- `模板名称`
- `版本号`（版本化）
- `生效时间/失效时间`（任务锁定模板版本）
- `检查项集合`（逐检查项配置）
  - `检查项名称`
  - `隐患类别`（7类之一）
  - `二级隐患选项`（可配置；含“其他”）
  - `二级其他自填字段`（如：自填隐患描述，必填策略）
  - `附件要求`（可选/必填/不要求）

**页面清单（中心执行）**
- `自查自纠任务列表`：本周任务周期、截止时间、任务状态（未开始/已提交/已关闭）
- `进入巡检`：展示任务信息 + 检查项填写
- `巡检记录提交详情`：提交内容回看（可配置是否可编辑）

**任务规则（按文档口径）**
- 每周根据《中心安全检查表》，至少完成一次中心场地安全巡检并上传巡检记录

### 巡检填写表单（逐检查项）
- `是否合格`（是/否，必填）
- 若 `是否合格=否`：
  - `二级隐患描述`（下拉，必填）
  - 若 `二级隐患描述=其他`：显示 `自填隐患描述`（必填）
  - `不符合描述`（必填，作为隐患的具体问题描述来源）
  - `附件`（按检查项配置：可选/必填）

### “生成隐患=必然”的映射规则（关键）
- 当检查项选择 `不合格/否` 时：**必生成 `隐患记录`**
- 粒度：`1个检查项结果 -> 1条隐患记录`（同一检查项拆分多条隐患请在模板侧拆分）
- 自动带出字段：
  - `来源`：巡检异常（自查自纠）
  - `所属任务`：本周巡检任务ID
  - `责任单位默认`：中心
  - `隐患类别`：取检查项配置的类别（7类）
  - `二级隐患描述`：取巡检选择
  - 若二级为“其他”：保存 `自填隐患描述`
  - `具体问题描述`：保存 `不符合描述`
  - `附件`：保存检查项上传附件
  - `状态`：`待稽核`

**提交后默认流转**
- 巡检提交后生成的隐患进入：`待稽核`

**提醒触发点**
- 巡检任务截止后仍为未提交：触发 `提醒推送`（见第3节）

## 2.3 总部稽核

**页面清单（总部/省区）**
- `总部稽核任务列表`：筛选（本月/状态/范围）、任务详情入口
- `稽核任务详情`：展示待稽核隐患列表；对每条隐患逐条稽核提交
- `隐患详情/整改单详情`：展示流转时间线、证据、整改与验收信息
- `申诉中心`：对稽核不通过/验收不通过等记录开放（按制度）

**稽核任务规则**
- `稽核任务下发`：月度全覆盖（总部创建并下发）
- `总部抽选省区复查`：子流程（先最小实现：抽选生成复查任务，复查结论用于最终证据/验收支撑）
- `稽核表单定期更新`：模板版本化，任务锁定模板版本

**稽核结果表单（总部逐条处理）**
- `稽核结论`：
  - `通过`（必然生成整改单）
  - `不通过`（走 B：退回修正 -> 再次稽核）
- `稽核原因/依据`（必填）

**结论触发的流转规则**
- 若 `稽核结论=通过`：
  - 隐患状态：`稽核通过（已生成整改单）`
  - 同步创建 `整改单`：状态 `待整改`
- 若 `稽核结论=不通过`（B）：
  - 隐患状态：`稽核不通过-待修正`
  - 不创建整改单（或按你们制度生成“修正指令”，但最终闭环仍落到“验收通过-关闭”）

**中心修正提交（承接 B）**
- 中心看到 `稽核不通过-待修正` 后提交：
  - `修正说明/补充材料`（必填）
  - `补充证据`（附件，可选/必填按制度）
- 提交后隐患状态：`待再次稽核`

**再次稽核**
- 回到总部稽核入口，对 `待再次稽核` 的记录继续稽核

**整改单验收闭环（最终关闭口径）**
- 整改单：`待整改 -> 整改中 -> 待验收 -> 验收通过-关闭`
- 若验收不通过：`验收不通过-退回整改` 回到整改中并再次提交验收

**申诉**
- 对象：稽核不通过（以及后续验收结果）可申诉（按制度开放范围）
- 字段：`申诉理由`（必填）、`申诉证据`（附件可选/必填按制度）
- 状态：`申诉中`（是否冻结再稽核按制度配置）

**完成时间要求**
- 稽核/修正/整改/验收节点均提供 `期限要求`
- 到期触发 `提醒推送` 与升级（第3节）
    
 
    

## 2.4 专项稽查（总部、省区）

**触发类型**
- 突发事件
- 国家要求
- 临时做表（临时配置检查项/稽核表单结构）

**页面清单**
- `专项稽查任务列表`
- `专项稽查任务详情`

**处理方式（复用2.3机制）**
- 专项稽查对“待查隐患/异常集合”执行：稽核 -> 通过生成整改单 -> 整改 -> 待验收 -> 验收通过-关闭
- 若稽核不通过：按 B 执行“退回修正 -> 再次稽核”
    
 
    

## 3、提醒推送功能
    
**目标**
- 对“未完成关键节点”的责任单位进行提醒；超阈值升级到主管并可触发通报。

**页面清单**
- `提醒规则设置`：配置阈值、触达次数、升级策略
- `提醒推送记录`：记录每次提醒与升级结果（用于审计与追溯）

**最少覆盖的触发规则（可配置）**
- 触发1：自查自纠任务截止后仍未提交
  - 触发对象：责任单位（默认中心）
  - 动作：提醒 -> 超阈值通报主管
- 触发2：整改单到期仍未提交验收
  - 触发对象：责任单位
  - 动作：提醒 -> 超阈值通报主管

**升级策略（最小实现）**
- `提醒1 -> 提醒2 -> 通报主管`
- 每次触达必须写入 `提醒推送记录`，保证可追溯
